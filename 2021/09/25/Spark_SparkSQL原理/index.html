Spark SQL执行原理

发表于 2021-09-25 分类于 Spark
概念 解释 产物
Parser
解析器		 将sql语句利用Antlr4进行词法和语法的解析。 语法树和Unresolved LogicalPlan。
仅仅是一种数据结构，不包含任何数据信息，例如不知道数据源，数据类型，不同的列来自哪张表等。
Analyzer
分析器		 进行规则解析。 Analyzed Logical Plan。
将表解析为具体的字段，确定数据来源，确定聚合函数等。
Optimizer
优化器		 优化逻辑计划。 Optimized Logical Plan。
谓词下推 + 列剪枝 + 常量替换 + 常量累加。
Planner
计划器		 生成多个物理计划，根据代价模型选择最优。 Physical Plan。
解析出数据源从JDBC读取，文件路径，数据类型等。
Code Generation
代码生成		 将 SQL 翻译成一系列的关系代数算子或表达式。
然后依赖这些关系代数算子逐条处理输入数据并产生结果。		 Code & DAG
在Driver端生成可执行代码和DAG有向无环图。
Execution
执行器		 执行代码。 计算结果。
