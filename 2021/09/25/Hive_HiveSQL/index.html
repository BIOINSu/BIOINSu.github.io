Hive SQL

发表于 2021-09-25 分类于 Hive
概念 解释
HiveSQL转MR（普通） 普通的任务直接用fetch的方式直接读取数据，不需要转MR。
HiveSQL转MR（Join） Map：生成键值对，以JOIN ON条件中的列作为Key，以JOIN之后所关心的列作为Value，在Value中还会包含表的 Tag 信息，用于标明此Value对应于哪个表。
Shuffle：将键值对发送至不同的Reducer中。
Reduce：Reducer通过 Tag 来识别不同的表中的数据，根据Key值进行Join操作。
HiveSQL转MR（Group By） Map：生成键值对，以GROUP BY条件中的列作为Key，以聚集函数涉及的字段作为Value。
Shuffle：将键值对发送至不同的Reducer中。
Reduce：根据SELECT子句的列以及聚集函数进行Reduce。
HiveSQL转MR（Distinct） 相当于没有聚集函数的GROUP BY，操作相同，只是键值对中的value可为空。
Hive SQL与Spark SQL 1、Hive SQL底层是MR，Spark SQL底层是RDD的计算。本质上两者的区别变成了MR和Spark的区别（内存、算子、容错）。

2、对于Hive本身来说，它包括metastore（元数据），就是schema的管理，比如有哪些表，哪些字段，什么数据类型，有没有索引等。而Spark只是一个计算引擎，没有元数据管理工具。

3、hive有hive on mapreduce和hive on spark。hive on spark 和Spark SQL并没有特别明显的性能优化（毕竟计算引擎都是Spark）。

4、hive on mapreduce：适合处理离线非实时数据，基于磁盘的处理使得性能上较为稳定。
Spark SQL：实时性要求或者速度要求较高的场所，基于内存的处理可能会出现数据倾斜造成反压等问题，导致内存溢出等错误。
