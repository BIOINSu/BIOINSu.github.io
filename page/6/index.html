MySQL读写分离与主从复制

发表于 2021-09-10 更新于 2021-09-13 分类于 数据库 ， MySQL
概念 解释
读写分离 将数据库分为主库和从库，主库只写数据，多个从库只读取数据。
缓解因写入操作时间过长而影响数据库查询效率的问题。
读写分离分机配置 1、程序代码内部实现：insert主库，select从库
2、中间代理服务器自动判断转发：MySQL Proxy连接池
主从同步延迟 1、二次读取，从库读不到就去读主库
2、写数据之后，立刻到来的读操作指定为访问主库
3、关键业务读写都由主库承担，非关键业务读写分离
主从复制 将主库写入、变更的数据同步到从库上。
使得从库提供和主库一致的数据。
MySQL支持的复制类型 1、基于语句，直接复制SQL命令。
2、基于行，直接复制数据。
3、混合方式，先语句后行。
主从复制过程 binlog复制并重放语句。

一致性哈希算法

发表于 2021-09-10 更新于 2021-09-11 分类于 分布式
概念 解释
一致性哈希算法 一致性哈希算法就是将哈希缓存的除数改为\(2^{32}\)，将映射空间看成一个hash环，把服务器和缓存数据看作是上面的每个点。
缓存数据会分配到沿顺时针方向遇到的第一个服务器上。
优点 服务器的数量如果发生改变，并不是所有缓存都会失效，而是只有部分缓存会失效。
hash环偏斜 服务器可能会不均匀地被映射到hash环上。
大量缓存数据可能会被映射到同一个服务器上。
虚拟节点 实际节点虚拟复制而来的节点，其实就是在hash环上再设置多个服务器的映射。
解决hash环偏斜导致缓存数据分配不均的问题。

虚拟内存

虚拟内存，是一种内存管理技术，为每个进程提供了⼀个连续私有的地址空间。可以把内存扩展到硬盘空间，使得程序认为它拥有连续的可用的内存（一个连续完整的地址空间），但实际上，它通常是被分隔成多个物理内存碎片，还有部分暂时存储在外部磁盘上，在需要时进行数据交换。

1、程序可以使⽤相邻的虚拟地址来访问物理内存中不相邻的内存缓冲区。

2、程序可以使⽤虚拟地址来访问⼤于可⽤物理内存的空间。

3、不同进程使⽤的虚拟地址彼此隔离，无法修改对方的物理内存。

分区

分区表是如何加速的

将数据按字段值划分开。

从文件上来看，每个分区就是HDFS上的一个目录。（可以指定多个分区形成多级分区）

查询时通过where进行分区过滤，查询指定分区的数据，避免全表扫描，加快查询速度。

分区表分区字段的指导原则

1、分区字段名称不能是表中已经存在的字段名称，因为分区字段最终也会以虚拟字段的形式显示在表结构上。

2、最好选用那些字段值分布均匀的字段来作为分区字段，否则单个文件可能会过大，失去了分区表的意义。

发表于 2021-09-06 更新于 2021-09-09 分类于 面试

计网

Syn攻击

是什么

TCP三次握手中，攻击者向服务器发送连接请求，服务器回复ACK确认后，攻击者并不返回ACK确认，即使得第三次握手无法完成，导致消耗服务器的资源，使得正常的SYN请求被丢弃。

怎么检测

当发现大量的大量的半连接状态时，特别是源IP地址是随机的时候，可以检测为是Sync攻击。

怎么防范

1、降低超时时间，使得主机尽快释放半连接的占用。

2、使用代理服务器。当客户端SYN包到达代理服务器时，SYN代理服务器并不转发SYN包，而是以服务器的名义主动回复SYN/ACK包给客户，如果收到客户的ACK包，表明这是正常的访问，此时代理服务器向服务器发送ACK包并完成三次握手。

Java内存模型

发表于 2021-09-05 分类于 Java ， Java原理
概念 解释
缓存一致性问题 各个CPU核中的缓存数据不一致导致计算结果不正确的问题。
Java内存模型（JMM） 是Java 定义的一套协议，用来屏蔽各种硬件和操作系统的内存访问差异，让Java 程序在各种平台都能有数据一致性的运行效果，解决缓存一致性的问题。
JMM协议的内容 1、所有的变量都存储在主内存中，每个线程还有自己的工作内存。

2、线程的工作内存中保存了该线程使用到的变量（主内存的拷贝），线程对变量的所有操作（读取、赋值）都必须在工作内存中进行，而不能直接读写主内存中的变量。

3、不同线程之间无法直接访问对方工作内存中的变量，线程间变量值的传递均需要在主内存来完成。
Volatile的作用 Java语言通过 volatile 修饰变量实现内存可见性，它强制修改后的变量写入主存，避免前后指令的CPU重排序，并及时让其他核中的相应缓存行失效，volatile是利用MESI达到符合预期的效果。
可见性 可见性是指当多个线程访问同一个共享变量时，一个线程修改了这个变量的值，其他线程能够立即看到修改后的值。

08 Flink反压

发表于 2021-09-04 更新于 2021-09-05 分类于 Flink
概念 解释
反压 某个计算节点的处理速度跟不上上游数据的发送速度，导致该节点成为计算瓶颈。
Flink 的网络传输架构 1、每个TaskManager内分为Flink、Netty、Socket三层的Buffer，向Buffer中写入数据。
2、TaskManager间采用TCP协议进行数据传输。
TCP-based反压机制 1、当缓冲区被耗尽之后，写入数据和读取数据的组件都分别停止工作，将停止发送数据的信号传递给上层直到Source端。
2、TaskManager间采用TCP协议的流控机制（滑动窗口）来告知发送方停止发送数据。
Credit-based反压机制 1、ResultSubPartition 向 InputChannel 发送消息的时候都会发送一个 backlog size 告诉下游准备发送多少消息。
2、下游计算有多少的 Buffer 去接收消息。
3、算完之后如果有充足的 Buffer 就会返还给上游一个 Credit 告知他可以发送消息。

环形缓冲区

发表于 2021-09-03 更新于 2021-09-16 分类于 Hadoop

概念

环形缓冲区是指数据被Map处理之后会先放入内存，内存中的这片区域就是环形缓冲区。

环形缓冲区其实是一个简单的buffer，即一个byte类型的数组，默认大小是100M。

当环形缓冲区达到容量的80%时，产生溢写（spill），数据从内存溢出到磁盘上。

阈值太小经常溢写导致IO操作影响性能，阈值太大影响内存并阻塞map的写操作。

数据库中键和范式

发表于 2021-09-01 更新于 2022-05-03 分类于 数据库

总结

概念 解释
键 指的是关系型数据库中一张表的某些字段
超键 在关系表中能唯一标识一行数据的字段集
候选键 不含有多余字段的超键被称为候选键，同样能够唯一决定一条数据
主属性 候选键中的字段
主键 用户选择的被用来唯一标识一行数据的的候选键被称为主键
范式 关系型数据库表需要满足的一些约束条件被称为范式。
一范式 字段不可再分
二范式 非主键字段全部依赖于主键（消除部份依赖）
三范式 非主键字段之间无依赖关系（消除传递依赖）
BCNF 消除主属性对候选键的部分和传递依赖
四范式 消除一张数据表内的多个一对多的关系
范式优点 消除冗余、异常
范式缺点 关联表增加查询复杂度，降低数据库查询性能
反范式化 在数据表中增加冗余字段来提升数据库的读性能。
